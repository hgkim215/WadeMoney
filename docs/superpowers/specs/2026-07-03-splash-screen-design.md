# WadeMoney 스플래시 화면 — 설계 스펙

Date: 2026-07-03
Status: Approved (브레인스토밍 완료, 구현 계획 대기)

## 배경 / 문제

`Info.plist`의 `UILaunchScreen`이 빈 `dict`로만 설정되어 있어 앱 실행 시 아무 커스터마이징 없는 밋밋한 화면이 잠깐 보인다. iOS 네이티브 런치스크린은 **정적 이미지만 지원**하며 코드가 전혀 실행되지 않아 애니메이션이 불가능하다 — 따라서 애니메이션이 있는 스플래시는 앱 프로세스가 뜬 직후 SwiftUI로 그린 커스텀 화면을 잠깐 보여주고 메인 화면(대시보드)으로 전환하는 방식으로 구현한다.

## 목표

- 앱 아이콘과 동일한 "도넛 먹는 돼지" 마스코트가 도넛을 실제로 베어무는 모션을 보여주는 스플래시 화면 추가.
- 콜드 스타트(앱을 완전히 종료했다가 다시 실행)에만 노출, 약 1.2~1.5초 후 자동으로 메인 화면 전환.
- 라이트/다크 모드 자동 대응, Reduce Motion 접근성 설정 대응.

## 비목표 (이번 스코프에서 제외)

- "WadeMoney" 워드마크/로고 텍스트 노출 — 마스코트 애니메이션만.
- 웜 스타트(백그라운드 복귀) 시 재노출.
- 스킵 제스처(탭해서 건너뛰기), 사운드 이펙트.
- Lottie 등 외부 애니메이션 라이브러리 도입.
- 실제 앱 초기화(데이터 로딩) 완료 시점에 맞춘 동적 길이 조정 — `WadeMoneyApp.init()`이 이미 동기적으로 끝나 있으므로 고정 시간 방식으로 충분.

## 결정한 접근법

기존 앱 아이콘 PNG를 통째로 애니메이션하는 방식은 도넛과 돼지가 한 장의 이미지라 "베어무는" 동작을 표현할 수 없어 제외했다. Lottie 등 외부 에셋 도입은 이 프로젝트에 새 의존성을 추가하는 부담에 비해 이득이 작아 제외했다.

**채택**: 지난 세션에 앱 아이콘을 생성할 때 쓴 벡터 좌표(200×200 좌표계)를 그대로 재사용해, 정적 PNG가 아닌 실제 SwiftUI `View`로 마스코트를 다시 그린다. 파츠(귀·눈·주둥이·도넛·부스러기)가 개별 View이므로 도넛의 위치·베어물림 진행도를 독립적으로 애니메이션할 수 있고, 최종 정지 프레임이 앱 아이콘과 픽셀 단위로 동일해 브랜드 일관성이 보장된다.

## 컴포넌트 구조

- **`MascotView`** (`WadeMoney/DesignSystem/MascotView.swift`)
  순수 벡터 View. 앱 아이콘 생성 스크립트와 동일한 도형/좌표/색상을 사용(아래 "마스코트 지오메트리" 참고). 다음 값을 외부에서 주입받아 애니메이션 가능한 상태로 노출한다:
  - `donutOffset: CGSize` — 도넛의 최종 위치(캔버스 기준 (158,124))로부터의 오프셋. 시작 값은 화면 밖 상단 방향(대략 (0, -90) 부근, 정확한 값은 구현 중 시각적으로 튜닝)이며 최종적으로 `.zero`로 수렴.
  - `donutRotation: Angle` — 접근 중 살짝 더 기울어져 있다가(-16°보다 큰 각도) 최종 -16°로 정착.
  - `biteMaskProgress: CGFloat` (0...1) — 0이면 도넛이 완전한 원, 1이면 아이콘과 동일한 베어문 자국(반지름 17.5pt, 도넛 로컬 좌표 (4,39) 고정 위치). 위치는 고정하고 반지름만 `17.5 * biteMaskProgress`로 애니메이션.
  - `mouthOpenProgress: CGFloat` (0...1) — 입 안쪽 타원(`#EE8FA0`)의 세로 스케일을 1.0→1.3→1.0으로 보간해 씹는 느낌을 낸다.
  - `faceScalePulse: CGFloat` — 돼지 얼굴 전체 그룹의 스케일(1.0 기준, 베어무는 순간 1.04까지 잠깐 커졌다 복귀).
  - `crumbs: [CrumbState]` (3개 고정) — 각각 opacity(0→1→0)와 offset(살짝 튀었다 떨어지는 궤적)을 독립적으로 가짐.

- **`SplashScreen`** (`WadeMoney/Screens/Splash/SplashScreen.swift`)
  `WadeColors.bg(colorScheme)` 배경 위에 `MascotView`를 얹고, 내부 `Task`로 아래 타임라인 순서대로 `withAnimation` 호출을 실행한다. 타임라인이 끝나면 `onFinished: () -> Void` 클로저를 호출한다. `@Environment(\.accessibilityReduceMotion)`이 켜져 있으면 개별 단계 애니메이션을 생략하고 최종 포즈로 즉시 페이드인한 뒤 훨씬 짧게(예: 0.4초) 대기하고 종료한다.

- **`RootView` 통합**
  `@State private var showSplash = true`로 시작. `ZStack`으로 `SplashScreen`을 `RootTabView` 위에 얹었다가, `onFinished`에서 `withAnimation { showSplash = false }`로 크로스페이드 전환. 별도의 영속 플래그 없이 프로세스 시작 시점의 초기값에 의존하므로, 앱을 완전히 종료 후 재실행할 때만 자연히 다시 나타난다(콜드 스타트 전용 요구사항을 별도 로직 없이 충족).

## 애니메이션 타임라인 (총 ~1.4초, Reduce Motion 비활성 기준)

| 구간 | 시간 | 동작 |
|---|---|---|
| 등장 | 0.0–0.35s | 마스코트만 opacity 0→1 페이드인(배경은 처음부터 완전히 보임 — 콜드 런치 직후 화면이 또 한번 비었다 채워지는 이중 블랭크를 피함). 최초 구현안(스케일 0.85→1.0)은 최종 리뷰에서 Reduce Motion 시에도 스케일 모션이 재생되는 문제가 발견되어 순수 페이드로 교체됨(branchless 설계 유지) |
| 도넛 접근 | 0.35–0.75s | `donutOffset`이 시작 오프셋→`.zero`, `donutRotation`이 최종 -16°로 정착 (ease-out 또는 spring) |
| 베어물기 | 0.75–0.95s | 도착과 동시에: `biteMaskProgress` 0→1 (spring, response 0.2~0.25, 살짝 오버슈트), `crumbs` 3개가 0.75/0.8/0.85s에 순차적으로 팝, `mouthOpenProgress` 0→1→0, `faceScalePulse` 1.0→1.04→1.0. **부스러기는 이후 정지 구간까지 사라지지 않고 계속 보인다** — 최초 초안은 팝 후 다시 0으로 페이드아웃하도록 되어 있었으나, 앱 아이콘(`icon-1024.png`)이 부스러기를 영구히 포함한 단일 정지 프레임이라는 사실과 모순되어 최종 리뷰에서 수정됨(정지 구간이 아이콘과 픽셀 단위로 일치해야 하므로 아이콘 쪽 요구가 우선) |
| 정지 | 0.95–1.2s | 최종 포즈로 홀드 (이 프레임이 앱 아이콘과 동일) |
| 퇴장 | 1.2–1.4s | `SplashScreen` 전체 opacity 1→0, 동시에 `RootTabView`가 아래에서 페이드인 |

## 접근성

- `@Environment(\.accessibilityReduceMotion)`이 `true`이면: 바운스/슬라이드/스프링 생략. 마스코트를 최종 포즈(베어문 상태)로 즉시 표시하고 짧게 페이드인 → 약 0.4초 대기 → 페이드아웃. 부스러기 팝/펄스 등 움직임 요소는 전부 생략.

## 검증 계획

이 작업은 로직보다 시각적 결과가 핵심이므로:
- 상태 전환 로직(스플래시 표시 → `onFinished` → 메인 화면 전환)만 최소 단위 테스트로 커버.
- 나머지는 iPhone 17e 시뮬레이터에서 스크린샷/화면 녹화로 육안 확인: 라이트/다크 × Reduce Motion on/off 총 4가지 조합.
- 기존 유닛 테스트(현재 117개)와 E2E(`CoreFlowUITests`)가 계속 통과해야 한다. 스플래시가 `testQuickAddExpenseFlowUpdatesHistory` 등 기존 E2E 테스트의 "한눈에" 텍스트 대기 로직과 충돌하므로, 테스트 환경(`XCTestConfigurationFilePath` 존재 시)에서는 `showSplash`를 처음부터 `false`로 초기화해 스플래시를 완전히 건너뛴다 — `WadeMoneyApp.init()`이 이미 테스트 호스트를 이 방식으로 감지해 인메모리 컨테이너로 우회하는 것과 동일한 패턴.
