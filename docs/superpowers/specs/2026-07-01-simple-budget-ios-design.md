# 간단한 가계부 iOS 앱 — 설계 문서

- **작성일**: 2026-07-01
- **상태**: 설계 승인됨, 구현 계획 대기
- **한 줄 요약**: 서버 없이 온디바이스로 동작하는 개인용 지출 가계부. 3탭 입력, 일/월/연 대시보드, "지난 기간 같은 시점 대비" 페이스 비교, 카테고리 시각화, 온디바이스 AI 서술을 제공한다.

---

## 1. 목표와 배경

빠르고 부담 없는 지출 기록과, 소비 패턴을 **한눈에** 파악하는 시각화를 목표로 한다. 핵심 사용자 요구는 다음과 같다.

1. **일/월/연**으로 구분해 지출 내역과 예산을 조회
2. **매우 간편한 지출 입력** UX
3. 어디에 소비했는지 **시각적으로 한눈에** 파악
4. 전월 대비 더/덜 썼는지 **즉각적으로** 파악
5. 어느 카테고리에 돈을 많이 쓰는지 **시각화**
6. (추가) 사용자가 아무것도 안 해도 되는 **수동적 AI 보조** — 메모 다듬기, 소비 통계·피드백

### 설계 원칙
- **간단함 우선(YAGNI)**: MVP 범위를 좁게 유지하고, 무엇을 안 만들지 명시한다.
- **프라이버시 완결성**: 데이터는 사용자의 iCloud 안에만 존재. 외부 서버·수집·API 호출 없음. AI도 온디바이스.
- **오프라인 우선**: 입력은 네트워크 없이 즉시 저장.
- **AI는 보조 레이어**: AI가 없어도(미지원 기기 포함) 앱은 완전히 성립한다.

## 2. 기술 스택

| 구성 | 선택 | 이유 |
|---|---|---|
| UI | **SwiftUI** | WidgetKit·App Intents·Swift Charts와 매끄럽게 연동, 네이티브 입력감 |
| 저장 | **SwiftData** (로컬 우선) | 오프라인 즉시 저장, 최신 iOS 표준 영속성 |
| 동기화 | **CloudKit(iCloud)** | SwiftData 플래그로 활성화. 서버·계정 시스템 불필요, 애플 ID 자동 |
| 차트 | **Swift Charts** | 도넛·막대·라인 네이티브 |
| AI | **Foundation Models** (온디바이스 LLM) | 무료·오프라인·데이터가 기기 밖으로 안 나감 |
| 최소 지원 | **iOS 26+** | Foundation Models 사용 |

### 전제 / 제약
- 통화는 **원화(KRW) 단일**. 다중 통화 없음.
- 데이터는 전부 사용자 iCloud 내부에만 저장. 외부 서버·수집 없음.
- **CloudKit 제약**: SwiftData+CloudKit는 모든 속성이 기본값 또는 옵셔널이어야 하고, 유니크 제약을 쓸 수 없다. 모델 설계에 반영.
- **Foundation Models 기기 제약**: Apple Intelligence 지원 기기에서만 동작. `SystemLanguageModel.availability`로 확인 후 미지원 시 AI 기능만 숨기고 나머지는 정상 작동(graceful degradation).

### 대안 검토(탈락)
- *React Native / Flutter*: 위젯·App Intent·Swift Charts 연동이 이질적이고 네이티브 입력감 손실, 서버리스 iCloud 동기화 손해.
- *UIKit*: 이 규모엔 과함. SwiftUI로 충분.

## 3. 기능적 정보 구조 (화면 면)

> 픽셀 레이아웃·탭 배치·네비게이션 형태는 **디자인 단계(클로드 디자인)에서 결정**한다. 아래는 "어떤 기능 면이 존재하고 무엇을 담으며 어떻게 연결되는가"만 규정한다.

### 3.1 입력 (Quick Add)
- 금액 키패드가 먼저·크게.
- 자주 쓰는 카테고리 버튼(사용 빈도순 자동 정렬).
- 메모(선택) + **`AI 다듬기`** 액션.
- 날짜: 기본 오늘, 탭하면 변경.
- 기본 **지출**, 상단에 작은 **수입 토글**(특수 수입용).
- 저장 시 햅틱 + 닫힘. 목표: **금액 → 카테고리 → 저장 3탭 이내**.

### 3.2 대시보드 (한눈에 조회)
- 상단 **일 / 월 / 연 전환** + 기간 이동(◀ ▶).
- 이번 기간 **총지출**.
- **예산 대비 진행**(소진율·남은 금액).
- **페이스 비교 배지**(지난 기간 같은 시점 대비 증감). → §5.2
- **AI 인사이트 카드**(지원 기기에서만).
- **카테고리 비중(도넛)** — 슬라이스 탭 → 해당 카테고리 내역으로 이동.
- **추세 미니 막대**(최근 기간 흐름).

### 3.3 내역 (List)
- 날짜별 그룹 리스트, 기간·카테고리 필터.
- 거래 수정/삭제.
- 특수 수입은 다른 색·부호로 표시(지출 차트·예산에는 불포함).

### 3.4 설정
- 월 예산 설정(단일 입력 칸).
- 카테고리 관리(추가/수정/색·아이콘/아카이브).
- AI 기능 on/off.
- iCloud 동기화 상태.
- CSV 내보내기.

### 3.5 앱 밖 진입점
- **홈/잠금화면 위젯**: 오늘 지출 + 남은 예산, 탭 시 빠른 입력으로 딥링크.
- **App Intent**: Siri/단축어/액션버튼으로 지출 기록("커피 4천원 기록" 식).

## 4. 데이터 모델

집계·페이스 비교·AI 인사이트는 **저장하지 않고 실시간 계산**한다. 저장 엔티티는 4개.

### `Transaction` (거래)
| 필드 | 타입 | 비고 |
|---|---|---|
| `id` | UUID | |
| `amount` | Decimal | 원 단위, 금액 오차 방지 |
| `type` | enum `.expense` / `.income` | 기본 `.expense` |
| `category` | → Category (옵셔널) | 지출은 필수, 수입은 없음 |
| `memo` | String? | AI 다듬기 대상 |
| `date` | Date | 사용자 지정(기본 오늘) |
| `createdAt` | Date | 정렬·감사용 |

### `Category` (카테고리)
| 필드 | 타입 | 비고 |
|---|---|---|
| `id` | UUID | |
| `name` | String | |
| `iconName` | String | SF Symbol |
| `colorHex` | String | 도넛/배지 색 |
| `sortOrder` | Int | 자주쓰는 순 정렬 |
| `isArchived` | Bool | 삭제 = 아카이브(소프트) |

- **삭제 정책**: 하드 삭제 대신 아카이브 → 입력 목록에선 사라지지만 과거 내역·통계엔 카테고리가 유지된다(과거 도넛 무결성 보존).
- **기본 시드**: 최초 실행 시 기본 카테고리 세트 생성(식비·교통·카페·쇼핑·의료·주거·문화·기타 등). 이후 사용자가 자유롭게 수정/추가.

### `MonthlyBudget` (월 예산 스냅샷)
| 필드 | 타입 | 비고 |
|---|---|---|
| `effectiveMonth` | 연-월 | 적용 시작 월 |
| `amount` | Decimal | 그 시점의 월 예산 |

- 예산을 바꾸면 **이번 달(이후) 레코드로 기록**한다.
- 특정 달의 예산 = "그 달 이하 중 `effectiveMonth`가 가장 최근인 레코드"의 값.
- **설정 UI는 "이번 달 예산" 입력 한 칸**만 노출; 스냅샷 관리는 내부 자동. → 과거 달의 "예산 대비"가 정확히 보존된다.

### `AppSettings` (단일 레코드)
| 필드 | 타입 | 비고 |
|---|---|---|
| `aiEnabled` | Bool | AI 기능 on/off |
| `monthStartDay` | Int | 월 시작일, 기본 1. 월급날 기준 집계용 |

## 5. 계산 & 시각화 로직

시각화의 "숫자"를 정확히 규정한다. 모든 집계는 **지출만**(수입 제외) 대상으로 하며 `Transaction`을 기간·카테고리로 쿼리해 실시간 합산한다.

### 5.1 기간 정의
`monthStartDay` 기준으로 경계를 잡는다(기본 1일이면 전부 달력 기준).
- **일**: 해당 하루.
- **월**: `[이번 달 시작일, 다음 달 시작일)`.
- **연**: 그 해의 12개 예산월 합(기본값이면 1/1~12/31).

### 5.2 예산 대비 진행
- **월**: `소진율 = 이달 지출합 ÷ 이달 예산`, `남은 = 예산 − 지출합`.
- **일**: `일예산 = 이달 예산 ÷ 이달 일수` → 그날 지출과 비교.
- **연**: `연예산 = 그 해 월예산 스냅샷들의 합` → 연 지출과 비교.
- 예산이 0/미설정이면 진행 링을 숨기고 지출액만 표시(0 나눗셈 가드).

### 5.3 페이스 비교 (시그니처 지표) — "지난 기간 같은 시점 대비"
- **경과 오프셋 D** = 기간 시작일부터 오늘까지의 일수. 진행 중이면 오늘까지, 완료된 기간이면 전체 길이.
- `현재누적 = 기간 시작 ~ 오늘까지 지출합`.
- `이전누적 = 직전 기간의 시작 ~ (그 기간의 D일째)까지 지출합`. 이전 기간이 더 짧으면 `min(D, 이전 기간 길이)`로 캡.
- `증감률 = (현재누적 − 이전누적) ÷ 이전누적`.
- **표시**: 더 씀 → 빨강 ▲, 덜 씀 → 초록 ▼.
- **적용 범위**: 월·연 뷰의 핵심 지표. 일 뷰는 페이스 대신 **일예산 대비**로 표현(하루짜리 페이스는 노이즈).
- **엣지**: 이전누적 = 0 또는 첫 기간이면 "비교할 이전 기록이 없어요"로 대체.

### 5.4 카테고리 비중 (도넛)
- 기간 내 지출을 카테고리별 합산, `비중 = 카테고리합 ÷ 총지출`.
- 내림차순, **상위 6개 + 나머지는 "기타"로 묶음**.
- 슬라이스 탭 → 해당 카테고리 내역으로 이동.
- 아카이브된 카테고리도 그 기간에 거래가 있으면 표시.

### 5.5 추세 미니 막대
- 월 뷰: 최근 6개 예산월 지출, 이번 달 강조.
- 연 뷰: 12개월.
- 일 뷰: 최근 7~14일.

### 5.6 예상 지출 (파생)
- `예상 = 현재누적 ÷ D × 이달 일수`. 페이스 스토리 및 AI 코멘트의 입력으로 사용(표시는 선택).

### 5.7 엣지 케이스
첫 기간(비교 불가) · 예산 미설정 · 데이터 0건 · 0 나눗셈 · 이전 기간이 더 짧음 — 모두 위 정의된 대체 표시로 처리한다.

## 6. AI 로직 (온디바이스, Foundation Models)

**핵심 원칙: "숫자는 Swift가 계산, 문장은 LLM이 서술."** 모든 수치는 앱이 정확히 계산하고, LLM은 그 요약을 자연어로 풀어주기만 한다(숫자 환각 방지).

### 6.1 소비 인사이트 코멘트
- **입력**: `{총지출, 예산, 소진율, 페이스 증감률, 상위 카테고리와 각 전월 대비 증감, 예상지출}`의 구조화 요약. 원거래 원문은 넘기지 않는다.
- **출력**: `@Generable`로 한국어 1~2문장 + 감정 태그(예: 경고/양호).
- **표시**: 대시보드 상단 인사이트 카드 또는 월말 요약.

### 6.2 메모 다듬기 / 카테고리 추천
- **입력**: 메모 원문.
- **출력**: `{정리된 메모, 추천 카테고리(기존 카테고리명 중)}`.

### 6.3 축소 동작
- `SystemLanguageModel.availability` 미지원 또는 `aiEnabled = false`이면 위 두 기능만 숨긴다. 나머지 앱 기능은 100% 정상.
- 생성은 비동기 — "생성 중" 상태를 처리한다.

## 8. 성공 기준 / 검증

- 지출 입력이 **금액 → 카테고리 → 저장 3탭 이내**.
- **계산 로직 정확성**(기간 경계·페이스 비교·예산 스냅샷 해석·도넛 그룹핑)을 **단위 테스트**로 검증한다. 이 로직 레이어가 테스트 가능한 핵심.

- AI 미지원 기기에서 핵심 기능 100% 정상.

## 9. 아키텍처 노트 (구현 가이드)

- 계산 로직(기간 집계, 페이스, 예산 스냅샷 해석, 도넛 그룹핑)은 SwiftData/뷰와 **분리된 순수 함수/서비스**로 두어 단위 테스트가 가능하게 한다.
- AI 레이어는 별도 서비스로 캡슐화하고, 가용성 체크·비동기·축소 동작을 한 곳에서 관리한다.
- 위젯·App Intent는 앱과 동일한 SwiftData 스토어(App Group)를 공유한다.
